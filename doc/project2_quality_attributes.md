# 프로젝트 2 - 품질 요소

> 미니 드라이브 시스템에 필요한 품질 요소 정리

---

## 📌 1. 사용자 기대 품질

사용자는 파일을 쉽고 빠르게 업로드하거나 다운로드할 수 있기를 기대한다.  
또한 필요한 파일을 빠르게 찾을 수 있어야 하며, 공유 기능이 직관적으로 동작해야 한다.  
파일이 안전하게 저장되고, 실수로 삭제한 경우 복구할 수 있는 기능도 중요하다.

---

## 🛠️ 2. 개발자 관점의 품질

개발자는 기능을 추가하거나 수정하기 쉬운 구조를 선호한다.  
따라서 코드의 가독성과 유지보수성이 중요하며, 기능별로 모듈화된 구조가 필요하다.  
또한 오류가 발생했을 때 원인을 빠르게 파악할 수 있도록 테스트와 로그 관리가 가능해야 한다.

---

## 🔐 3. 관리자 관점의 품질

관리자는 시스템이 안정적으로 운영되기를 기대한다.  
사용자 계정 및 권한을 효율적으로 관리할 수 있어야 하며, 외부 공유 시에도 보안이 유지되어야 한다.  
또한 저장 공간과 사용자 수가 증가하더라도 성능이 크게 저하되지 않아야 한다.

---

## ✅ 4. 핵심 품질 요소 요약

| 품질 요소 | 설명 |
|----------|------|
| 사용성 | 업로드, 다운로드, 검색, 공유가 쉬워야 함 |
| 성능 | 검색 및 다운로드가 빠르게 동작해야 함 |
| 보안성 | 권한 제어, 인증, 안전한 공유가 가능해야 함 |
| 유지보수성 | 기능 수정 및 확장이 쉬운 구조여야 함 |
| 신뢰성 | 파일 손실 없이 안정적으로 동작해야 함 |
