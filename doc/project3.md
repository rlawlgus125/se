# 📌 프로젝트 관리 계획서

## 1. 문서 개요

본 문서는 프로젝트 진행을 위한 관리 계획서입니다.  
프로젝트의 **목적, 개발 계획, 팀 구성, 품질 관리, 개발 환경, 산출물 관리, 리스크 관리** 내용을 정리합니다.

---

## 2. 프로젝트 개요

### 프로젝트명

**누구나 확인 가능한 Open CCTV**

### 프로젝트 목적

CCTV를 누구나 쉽게 설치할 수 있도록 저렴한 디바이스 패키지를 제작하고,  
설치된 CCTV 영상을 웹페이지에서 실시간으로 확인할 수 있는 서비스를 제공합니다.

### 주요 기능

| ID | 기능 설명 |
|---|---|
| F01 | 라즈베리파이와 카메라 모듈을 이용한 화면 캡처 |
| F02 | 화면 스트리밍 서버 프로그램 제작 |
| F03 | 설치된 CCTV의 녹화 정보 DB 저장 |
| F04 | CCTV 위치를 구글/네이버 지도 등에 표시 |
| F05 | CCTV 선택 시 실시간 화면 제공 |

## 3. 개발 계획 🛠️

### 개발 절차 모형

요구사항 변경 가능성이 높기 때문에  
**Agile 방법론 중 Scrum 모델**을 사용합니다.

### 개발 방법론

객체지향 방법론을 사용하여  
분석, 설계, 구현, 테스트, 유지보수 및 확장성을 고려합니다.

### 주요 개발 활동

| 개발 활동 | 설명 |
|---|---|
| 프로젝트 요약 | 프로젝트 요약서 작성 |
| 프로젝트 계획 | 프로젝트 계획서 작성 |
| 요구사항 정의 | 요구사항 정의서 작성 |
| 요구사항 분석 | 요구사항 분석서 작성 |
| Product Backlog | 요구사항을 우선순위대로 정리 |
| Sprint Planning Meeting | 스프린트 시작 시 백로그 선정 및 계획 |
| Daily Scrum Meeting | 매일 진행상황과 이슈 공유 |
| Sprint | 1~4주 단위로 개발 및 피드백 진행 |
| Sprint Retrospective | 개선할 점, 잘한 점, 새로 시작할 점 도출 |
| Sprint Review | 스프린트 종료 시 작업 리뷰 및 문서화 |

## 6. 팀 구성 👥

### 팀 구조

본 프로젝트는 팀장의 역할은 존재하지만,  
모든 팀원이 상호 협력하는 **수평적 팀 구조**를 따릅니다.

| 역할 | 책임 |

| 웹 관리 | 웹페이지 개발 및 관리 |

## 7. 품질 관리 ✅

### 팀 미팅 계획

| 회의 | 일정 | 주요 내용 | 참석자 |
|---|---|---|---|
| Sprint Planning Meeting | 매주 금요일 18:00 | Sprint 목표 및 Backlog 계획 | 팀원, 팀장 |
| Daily Scrum Meeting | 매일 18:30 | 어제 한 일, 오늘 할 일, 장애 현상 공유 | 팀원, 팀장 |
| Sprint Review | Sprint 종료 시점 | Sprint 기간 동안 각자 한 일 리뷰 | 팀원, 팀장 |
| Sprint Retrospective | Sprint 종료 시점 | 개선할 점, 잘한 점, 새로 시작할 점 도출 | 팀원, 팀장 |
| 비정기 회의 | 필요 시 | 프로젝트 진행 중 문제 발생 시 진행 | 팀원, 팀장, 지도교수, 멘토 |

### 변경사항 관리

- 기능 변경은 팀원 전체가 권한을 가지며, 다수결의 합의가 필요합니다.
- 변경사항이 발생하는 경우 변경 전후 내용을 반드시 기록합니다.
- 관련 산출물은 팀원이 전부 소지합니다.
- 의견 충돌로 진행이 멈출 경우 팀장이 더 강한 발언권을 가집니다.
- 변경사항이 결정되면 지도교수에게 반드시 보고합니다.

### 산출물 관리

| 구분 | 규칙 | 예시 |
