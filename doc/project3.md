# 🔄 프로젝트 3 - 프로세스 모델

> 미니 드라이브 시스템 개발을 위한 프로세스 모델 선정

---

## 📌 1. 시스템 특성

본 시스템은 파일 관리 및 공유 기능을 포함하는 웹 기반 서비스이다.  
유사한 시스템(Google Drive, Dropbox 등)이 이미 존재하며, 기능이 점진적으로 확장될 가능성이 높다.  
또한 사용자 요구사항이 개발 과정에서 변경될 수 있는 특징을 가진다.

---

## ⚙️ 2. 프로세스 모델 선택

애자일(Agile) 모델을 선택한다.

---

## 🧠 3. 선택 이유

애자일 모델은 요구사항 변경에 유연하게 대응할 수 있으며,  
기능 단위로 반복적인 개발과 개선이 가능하다.  
사용자 피드백을 반영하여 점진적으로 시스템을 발전시킬 수 있다는 점에서 본 프로젝트에 적합하다.

---

## 🔁 4. 개발 흐름

1. 요구사항 정의  
2. 기능 단위 설계  
3. 반복 개발 (Iteration)  
4. 테스트 및 피드백 반영  
5. 기능 개선 및 확장  

---

## 🎯 5. 기대 효과

- 요구사항 변화에 유연하게 대응 가능  
- 빠른 기능 개발 및 개선  
- 사용자 중심
