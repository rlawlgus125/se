# 프로젝트 관리 계획서

## 1. 문서 개요

본 문서는 프로젝트 진행을 위한 관리 계획서이다.  
프로젝트의 목적, 개발 계획, 팀 구성, 품질 관리, 개발 환경, 산출물 관리, 리스크 관리 내용을 정리한다.

## 2. 프로젝트 개요

### 프로젝트명
누구나 확인 가능한 Open CCTV

### 프로젝트 목적
CCTV를 누구나 쉽게 설치할 수 있도록 저렴한 디바이스 패키지를 제작하고,  
설치된 CCTV 영상을 웹페이지에서 실시간으로 확인할 수 있는 서비스를 제공한다.

### 주요 기능
| ID | 기능 설명 |
|---|---|
| F01 | 라즈베리파이와 카메라 모듈을 이용한 화면 캡처 |
| F02 | 화면 스트리밍 서버 프로그램 제작 |
| F03 | 설치된 CCTV의 녹화 정보 DB 저장 |
| F04 | CCTV 위치를 구글/네이버 지도 등에 표시 |
| F05 | CCTV 선택 시 실시간 화면 제공 |

## 3. 개발 계획

### 개발 절차 모형
본 프로젝트는 요구사항 변경 가능성이 높기 때문에 Agile 방법론 중 Scrum 모델을 사용한다.

### 개발 방법론
객체지향 방법론을 사용하여 분석, 설계, 구현, 테스트, 유지보수 및 확장성을 고려한다.

### 주요 개발 활동
- 프로젝트 요약
- 프로젝트 계획
- 요구사항 정의
- 요구사항 분석
- Product Backlog 작성
- Sprint Planning Meeting
- Daily Scrum Meeting
- Sprint Review
- Sprint Retrospective

## 4. WBS

프로젝트는 다음과 같은 작업 단위로 구분된다.

- 요구분석
  - 계획서 검토
  - 기존 CCTV 조사
  - 요구 분석 및 정의
  - 필요한 기능 정의

- 설계
  - 분석서 검토
  - 시스템 구조 설계
  - 상세 설계

- 코딩
  - 기능 구현
  - 서버 및 DB 구축
  - 각 부분 기능 통합

- 테스트
  - 전체 테스트
  - 매뉴얼 작성
  - 발표 자료 작성

## 5. 팀 구성

본 프로젝트는 팀장의 역할은 존재하지만, 모든 팀원이 상호 협력하는 수평적 팀 구조를 따른다.

| 역할 | 책임 |
|---|---|
| 팀장 | 프로젝트 전반 진행 및 검토 |
| 문서관리 | 산출물 및 문서 관리 |
| 자료조사 | 프로젝트 관련 자료 조사 |
| DB 관리 | 데이터베이스 구축 및 관리 |
| 웹 관리 | 웹페이지 개발 및 관리 |

## 6. 품질 관리

### 팀 미팅
- Sprint Planning Meeting: 매주 금요일 18:00
- Daily Scrum Meeting: 매일 18:30
- Sprint Review: Sprint 종료 시점
- Sprint Retrospective: Sprint 종료 시점
- 비정기 회의: 프로젝트 진행 중 문제 발생 시 진행

### 변경사항 관리
- 기능 변경은 팀원 전체 합의를 통해 결정한다.
- 변경 전후 내용을 기록한다.
- 변경사항 확정 후 지도교수에게 보고한다.

### 산출물 관리
- 소프트웨어: GitHub 저장소에 관리
- 문서: 공유 클라우드 서버에 관리
- 기타 파일: 목적이 드러나는 파일명으로 관리
