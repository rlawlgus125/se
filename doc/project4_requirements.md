# 📌 요구사항 정의서

## 1. 서론

### 1.1 문서의 목적 및 범위

본 문서는 **Mini Drive 시스템**에서 요구되는 기능을 정의하기 위한 문서이다.  
파일 관리 및 협업 시스템에서 필요한 **기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항**을 정의한다.

---

### 1.2 대상 시스템 개요

#### 1.2.1 시스템 정의

Mini Drive는 조직 내 파일을 중앙에서 관리하고 사용자 간 공유를 지원하는  
**웹 기반 클라우드 파일 관리 시스템**이다.

사용자는 웹 브라우저를 통해 별도의 설치 없이 파일을 업로드, 다운로드, 공유할 수 있으며  
권한 기반 접근 제어 및 버전 관리 기능을 제공한다.

#### 1.2.2 주요 기능 설명

| ID | 설명 |
|---|---|
| F01 | 파일 업로드 및 다운로드 |
| F02 | 폴더 생성 및 관리 |
| F03 | 사용자 간 파일 공유 |
| F04 | 파일 검색 및 필터링 |
| F05 | 파일 버전 관리 |
| F06 | 사용자 인증 및 권한 관리 |

---

### 1.3 용어 정의

| 용어 | 설명 |
|---|---|
| 클라우드 스토리지 | 인터넷 기반 파일 저장 시스템 |
| 버전 관리 | 파일 변경 이력을 저장하고 이전 상태로 복원하는 기능 |
| 권한 관리 | 사용자별 접근 권한을 설정하는 기능 |
| API | 시스템 간 기능을 연결하는 인터페이스 |
